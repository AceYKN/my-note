## 术语解释

### Relation

关系是域的笛卡尔积的子集。

### Referential Integrity

参照完整性：若属性或属性组 `F` 是基本关系 `R` 的外码，它与基本关系 `S` 的主码 `Ks` 相对应，则对于 `R` 中每个元组在 `F` 上的值必须满足：

1. 取空值，即 `F` 的每个属性值均为空值；或
2. 等于 `S` 中某个元组的主码值。

### Role

角色是数据库中一组权限的集合。

### ACID Properties

事务的 ACID 特性：

- Atomicity：原子性。
- Consistency：一致性。
- Isolation：隔离性。
- Durability：持久性。

### Dirty Data

脏数据：假设事务 A 更新了一行数据，事务 B 读取了该行的新值；之后事务 A 回滚，则事务 B 读到的就是未提交且后来被撤销的数据，即脏数据。

## 判断题与参考答案

题面：

1. 一个基本表只能建立一个聚簇索引。
2. 数据库的并发操作一定会出现异常，因此必须实行封锁机制。
3. 突然停电属于介质故障。
4. `NULL` 值代表了空字符串。
5. 嵌入式 SQL 与主语言通信是靠 SQLCA 和变量来实现。
6. MAC 中主体和客体的密级通过对比来确定能否访问数据。
7. 所有的视图都是可更新的。
8. 关系的主码和外码中的属性均不能取空值。
9. “年龄限制在 16-25 岁之间”约束属于 DBMS 的安全性功能要求的实现。
10. 查询优化时，连接运算应尽可能先做。

参考答案与理由：

1. 对。聚簇索引决定表中数据物理排列方式，一个表通常只能有一个。
2. 错。并发操作可能出现异常，不是一定出现；封锁机制是为了预防并发异常。
3. 错。突然停电通常属于系统故障，不是介质故障。
4. 错。`NULL` 表示未知、缺失或不适用，不等同于空字符串。
5. 对。嵌入式 SQL 通过 SQLCA 和主变量与宿主语言通信。
6. 对。强制访问控制 MAC 依赖主体/客体安全级别或密级比较。
7. 错。复杂视图、含聚合/分组/连接等视图通常不可更新。
8. 错。主码属性不能取空值；外码属性在允许时可以为空。
9. 错。这是完整性约束，不是安全性功能。
10. 错。查询优化通常尽量先做选择和投影，减少中间结果；连接顺序需优化，不能简单“尽可能先做”。

## 选择题与参考答案

### 1. 规范化目的

题面：以下四个叙述中，哪一个不是对关系模式进行规范化的主要目的？

A. 减少数据冗余
B. 解决更新异常问题
C. 加快查询速度
D. 在模式设计上实现较好的语义表达/单一化

答案：C。

方法：规范化主要解决冗余、插入异常、删除异常、更新异常和语义不清；性能提升不是主要目的，有时规范化还会增加连接成本。

### 2. 教师只能访问自己课程成绩

题面：对于学生、选课、课程三类关系，若要求“每个教师只能存取自己讲授课程的学生成绩”，应建立什么机制？

可见选项包括：A. 视图；C. 游标；D. 索引。

答案：A. 视图。

方法：按教师过滤课程与成绩，给教师授权访问对应视图，而不是直接访问基本表。

### 3. 现实世界的“特征”对应数据世界的什么

题面：现实世界的“特征”对应于数据世界的（）。

A. 属性
B. 联系
C. 关系/记录，原图局部不清
D. 数据项

答案：A. 属性。

### 4. X 锁权限

题面：如果事务 T 获得了数据项 Q 上的 X 锁，则 T 对 Q（）。

A. 只能读不能写
B. 只能写不能读
C. 既可读又可写
D. 不能读不能写

答案：C。

### 5. SQL 空值操作中不正确的是

题面：SQL 中，空值操作中不正确的是（）。

A. `NOT (SAGE IS NULL)`
B. `SAGE != NULL`
C. `SAGE IS NOT NULL`
D. `SAGE IS NULL`

答案：B。

方法：SQL 判断空值必须用 `IS NULL` 或 `IS NOT NULL`，不能用 `= NULL` 或 `!= NULL`。

### 6. 取出某些列并消去重复元组

题面：取出关系中的某些列，并消去重复元组的关系代数运算称为（）。

A. 取列运算
B. 投影运算
C. 连接运算
D. 选择运算

答案：B。

### 7. 全码关系模式最高可达到的范式

题面：全码关系模式最高可以达到的范式为（）。

A. 1NF
B. 2NF
C. 3NF
D. BCNF

答案：D。

### 8. 脏数据

题面：“脏数据”是指（）。

可见选项：

A. 未提交的数据
B. 未提交的后又被撤销的数据
C. 违反访问权而写入 DB 的数据
D. 输入时就有错的数据

答案：B 更贴近题面；A 是概括性描述。

方法：脏读的关键是读取了其他事务未提交、并可能回滚的数据。

### 9. 关系运算中耗时操作

题面：关系运算中花费时间可能很长的运算是（原图底部不完整）。

参考答案：连接或笛卡尔积。若题目强调查询优化中最应避免的大中间结果，通常选笛卡尔积。

## NULL、完整性与补考名单例子

可见课堂内容：

```sql
-- Dormno is NULL
-- isNULL() / setNull()
SELECT *
FROM student
WHERE dormno IS NULL;

SELECT *
FROM student
WHERE dormno IS NOT NULL;

-- 下学期补考名单
SELECT *
FROM grade
WHERE score < 60 OR score IS NULL;

-- 用户自定义完整性示例
CHECK (sex = '男' OR sex = '女')
CHECK (sage >= 1 AND sage <= 150)
```

要点：

- 参照完整性可配合 `SET NULL` 策略。
- 用户自定义完整性常用 `CHECK` 约束表达。
- `score IS NULL` 与低于 60 分都可能进入补考名单，具体业务规则以教师说明为准。
