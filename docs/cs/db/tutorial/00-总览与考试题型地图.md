# 数据库系统完整讲义总览与考试题型地图

这套讲义按本课程近年考试和课堂投影整理，不按教材目录机械复述，而是按“考试会怎么问、答题时怎么写、为什么这样写”组织。主要参考资料包括：王珊等《数据库系统概论》第 6 版、2020-2024 历年期末题、2025-2026 新上传 `new-pic` 投影/板书、`pic整理文本` 课堂材料、补充习题 70+100 题。

## 一、近年题型结构

### 中文卷常见结构

常见分布为：

- 选择题：数据库基础概念、关系模型、SQL 语法、完整性/安全性、事务、封锁、恢复。
- 填空题：教材中的精确定义、三层模式、数据模型三要素、SQL 组成、完整性分类、恢复/并发术语。
- 简答题：DBMS 功能、数据库系统优点、视图优点、完整性与安全性区别、事务 ACID、死锁、恢复策略。
- SQL 题：建表、约束、索引、视图、查询、聚合、嵌套查询、删除/更新、授权。
- 数据库设计题：E-R 图、E-R 转关系模式、主码外码、函数依赖、范式判断、异常、分解到 3NF/BCNF。
- 数据库编程题：存储过程、触发器、游标、嵌入式 SQL、事务控制。
- 查询优化题：写关系代数、画原始查询树/语法树、做等价变换、画优化树、写优化后的关系代数。
- 恢复与并发题：检查点 UNDO/REDO、事务调度、封锁协议、两阶段封锁、死锁判断。

### new-pic 最新样题结构

新图里出现了更明确的专题权重：

- Explain the glossaries / Fill in the blanks：关系、参照完整性、角色、ACID、脏数据等术语。
- SQL：SaleDetail/Product/Stock 这类公司数据库，要求建表、级联更新、禁止级联删除、删除某日期后销售明细、插入产品、统计 2023 销售额、涨价、建视图、建索引、建用户并授权。
- Database programming：营业时间触发器、带参数存储过程、游标循环、输出参数。
- Relational Database Design：E-R 图、关系模式、主码外码、范式说明。
- Query Optimization：用笛卡尔积写等价关系代数，画原始树，给优化过程，写优化表达式。
- Database Recovery：检查点恢复，判断哪些事务 UNDO、哪些 REDO。
- Concurrency Control：给事务加锁/解锁，满足两阶段封锁协议，并判断是否可能死锁。

## 三、最重要的得分策略

大题不要只写结果，要写“依据”。数据库设计题要写实体、联系、主码、外码、范式理由；查询优化题要写原始表达式、原始树、优化规则、优化树；恢复题要写判断规则；编程题要写参数、条件判断、异常/不存在处理、游标关闭释放或事务提交回滚。

填空题不要凭感觉写同义词。教材里很多术语有固定表述，例如：

- 数据模型三要素：数据结构、数据操作、完整性约束。
- 数据库系统核心：DBMS。
- 数据库与文件系统本质区别：数据结构化。
- 数据库系统特点：数据结构化、共享性高/冗余低、独立性高、由 DBMS 统一管理。
- 关系完整性：实体完整性、参照完整性、用户自定义完整性。
- 恢复冗余技术：数据转储、登记日志文件。
- 基本封锁：共享锁 S、排他锁 X。
- SQL 综合语言：DDL、DML、DCL，有时教材也把查询语言 DQL 单列。

## 四、考场时间分配

如果是 120 分钟闭卷，建议：

- 选择填空：20-25 分钟。不会的先标记，后面凭大题知识回填。
- SQL：20 分钟。先写能确定的基本语句，复杂嵌套最后补。
- 设计/规范化：25-30 分钟。先列实体和函数依赖，防止画图卡太久。
- 编程：15-20 分钟。先写结构：参数、声明、判断、核心 SQL、提交/关闭。
- 优化/恢复/并发：20 分钟。严格按模板写，树可以画简洁文本结构。
