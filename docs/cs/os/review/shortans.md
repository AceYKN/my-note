# 简答题

**1．为什么分页技术不能实现共享，而分段技术可以？**
分页技术对程序员是透明的，系统强制将进程划分为固定大小的页，一页中往往混合了无关的代码和数据，这使得基于逻辑意义的共享和权限保护非常困难。而分段技术对程序员可见，程序被划分为具有逻辑意义的段（如完整的某个函数段、独立的数据表）。由于段是一个逻辑实体，操作系统只需要让多个进程的段表指向同一个物理段内存地址，即可轻松实现程序的共享，且易于设定访问权限（保护）。

**2．引入缓冲的目的。**
引入I/O缓冲（Buffering）的目的是为了提高操作系统的整体效率，解决设备瓶颈问题。缓冲机制可以缓解CPU与I/O设备之间处理速度不匹配的矛盾，通过吸收快速的数据突发（rapid bursts of I/O）来减少CPU的空闲等待时间，从而提高处理器和I/O设备之间的并行工作能力。

**3．什么是文件的物理结构？有哪几种组织方式？**
文件的物理结构（磁盘布局）是指文件在辅助存储器（如磁盘）上的实际存储与组织形式。主要有三种分配/组织方式：
1. **连续分配（Contiguous allocation）：** 文件在创建时分配一组连续的磁盘块。
2. **链式分配（Chained allocation）：** 按块分配，每一块包含指向下一块的指针。
3. **索引分配（Indexed allocation）：** 为每个文件建立一个索引表，存放分配给该文件的所有物理块号。

**4．分析操作系统特征。**
操作系统作为连接硬件与应用程序的接口，具有以下三个核心设计特征（目标）：
1. **方便性（Convenience）：** 隐藏硬件复杂细节，使用户更方便地操作计算机。
2. **高效性（Efficiency）：** 能够以高效的方式分配和管理系统资源（如处理器、内存和I/O）。
3. **可演进能力（Ability to evolve）：** 具有模块化结构，能够在不干扰现有服务的情况下方便地开发和引入新功能。

**5．试述临界区访问遵循的原则。**
为确保互斥与系统安全，临界区的访问必须遵循以下原则：
1. **互斥（Mutual exclusion）：** 一次只允许一个进程进入其临界区。
2. **空闲让进：** 临界区无进程时，任何请求进入的进程应被立即允许，无死锁或饥饿现象。
3. **有限等待：** 进程在临界区内停留的时间必须是有限的。
4. **让权等待（可选/推论）：** 发生在非临界区停止的进程绝不能干涉或阻塞其他进程。
5. 不对处理器的相对执行速度或处理器数量做任何假设。

**6．什么是文件的逻辑结构？有哪几种组织方式？**
文件的逻辑结构是从用户和应用程序的角度看到的数据组织形式。通常分为五种：
1. **堆文件（Pile）：** 数据按到达的顺序被收集，没有特定结构。
2. **顺序文件（Sequential file）：** 记录按某个关键字段严格顺序排列。
3. **索引顺序文件（Indexed sequential file）：** 在顺序文件基础上增加索引表和溢出文件以支持随机访问。
4. **索引文件（Indexed file）：** 完全通过索引访问记录，支持变长记录。
5. **直接或散列文件（Direct/Hashed file）：** 利用散列（Hash）函数直接计算记录物理地址进行极速访问。

**7．试述设备管理的基本功能。**
设备管理（I/O管理）的基本功能是在保证系统效率（Efficiency）和通用性（Generality）的前提下，实现应用与硬件底层设备的解耦。主要包括：管理逻辑I/O与物理设备之间的转换，分配和释放I/O通道及设备，处理I/O中断，管理I/O缓冲区，以及执行磁盘调度策略等。

**8．分区存储管理技术有哪些分配策略？**
在动态分区存储管理中，系统通过维护空闲内存块列表来装入进程，常见的分配/放置策略有：
1. **最佳适配（Best-fit）：** 选择大小与请求最接近的空闲块。
2. **首次适配（First-fit）：** 从内存头部开始扫描，选择第一个满足大小的空闲块。
3. **下次适配（Next-fit）：** 从上次放置的位置开始扫描寻找满足大小的块。
4. **最差适配（Worst-fit）：** 选择当前内存中最大的空闲块进行分配。

**9．什么是文件和文件系统？**
**文件（File）：** 是相似或相关数据记录的集合，被用户和应用程序视为一个单一的逻辑实体，且可通过名称被引用。
**文件系统（File System）：** 是操作系统中管理文件的系统软件集合，负责为用户和应用程序提供关于文件的创建、定位、访问控制、共享与存储服务。

**10．为什么要采用虚拟存储器管理，实现虚拟存储器的条件是什么？**
**原因：** 虚拟内存能够解除进程必须完全装入物理内存的限制，允许执行大于主存的程序（扩大了地址空间）。同时，由于只需调入部分页面，内存可以驻留更多进程，极大提高了多道程序的并发度和CPU的利用率。
**实现条件：**
1. **硬件支持：** 处理器需配备地址转换硬件（如MMU），支持分页或分段的动态地址映射，并能在发生越界或缺页时抛出中断。
2. **OS软件支持：** 操作系统需具备处理缺页中断（Page Fault）、页替换算法及辅存交换管理的软件机制。

**11．内存利用率不高主要表现在哪些方面？可通过哪些途径来提高内存利用率？**
主要表现在两种内存碎片：
1. **内部碎片（Internal fragmentation）：** 固定分区或分页系统中，分配给进程的块大于实际需求，块内部空间浪费。
2. **外部碎片（External fragmentation）：** 动态分区或分段系统中，内存中存在许多无法装入新进程的细小空闲块。
**提高途径：** 采用动态分区时，利用**压缩（Compaction/移动技术）**把碎片拼接成大块；采用**分页（Paging）**技术以消除外部碎片；采用**虚拟存储机制**根据需要按需调页，不将整个进程全加载入内存。

**12．什么是高级调度、中级调度和低级调度？**
1. **高级调度（Long-term scheduling）：** 也称作业调度，决定将哪些新创建的进程加入可执行进程池，控制系统的多道程序设计度。
2. **中级调度（Medium-term scheduling）：** 负责主存与辅存之间的换入换出（Swapping），决定哪些进程部分或全部应被挂起至辅存，以管理内存压力。
3. **低级调度/短程调度（Short-term scheduling）：** 也称进程调度，负责决定处理器下一个应该执行哪个就绪队列中的进程。

**13．什么是设备独立性，它是如何实现的？**
**设备独立性（Generality/通用性）：** 是指操作系统向程序隐藏了特定硬件设备的具体细节，使得用户程序能以统一的标准I/O指令处理各类设备，而无需关心它们是磁盘、磁带还是显示器。
**实现方式：** 操作系统引入了层次化的I/O逻辑模块。最底层由特定的**设备驱动程序（Device drivers）**和硬件抽象层处理，而上层向应用暴露统一的文件或流API。

**14．进程的基本状态有哪些？这些状态之间是如何转换的？**
五态模型包含：新建态（New）、就绪态（Ready）、运行态（Running）、阻塞态（Blocked）和退出态（Exit）。
**状态转换：**
*   **New → Ready：** OS接纳新进程。
*   **Ready → Running：** 调度程序选中该进程执行（Dispatch）。
*   **Running → Ready：** 时间片用完或被高优先级进程抢占（Timeout/Preempt）。
*   **Running → Blocked：** 进程请求I/O或等待其他事件（Event Wait）。
*   **Blocked → Ready：** 等待的I/O完成或事件发生（Event Occurs）。

**15．SPOOLing的含义是什么？试述SPOOLing系统的特点、功能以及控制过程。**
**SPOOLing**（Simultaneous Peripheral Operations On-Line，假脱机操作）是一种在多道程序环境中将独占设备改造为共享设备的技术。
**特点与功能：** 它利用磁盘作为巨大的缓冲区空间，缓解了主机与慢速设备速度不匹配的矛盾，提高了I/O并行性。
**控制过程：** 系统由输入进程、用户进程和输出进程构成。当用户进程需要使用打印机时，并不直接分配物理打印机，而是将输出数据发送到辅存（磁盘）上的输出缓冲区排队；独立的输出进程（Spooler）会根据设备空闲情况，逐个将缓冲区的数据送往实际的外部设备。

**16．简述进程的几种状态和引起状态转换的典型原因，以及相关的操作原语。**
同第14题。操作原语涉及：调度执行的 `Dispatch`，因请求资源受阻的 `Block/Wait`，因I/O完成被唤醒的 `Wakeup/Signal`，因时间片耗尽被迫让出CPU的 `Preempt`，以及销毁的 `Exit` 等。

**17．设备管理中的数据传送控制方式有哪几种？分别简述如何实现的。**
1. **编程式 I/O（Programmed I/O）：** 处理器代表进程发出I/O指令后，陷入盲等待（Busy-wait）状态，不断轮询设备直到完成。
2. **中断驱动 I/O（Interrupt-driven I/O）：** 处理器发出I/O指令后继续执行程序的后续指令。当设备完成I/O时，发送中断信号，处理器随即响应中断进行处理。
3. **直接内存访问（DMA）：** 设置专用的DMA模块控制主存与I/O间的数据交换。处理器发送传输请求后立刻放权，由DMA模块全权负责整块数据传输，仅在传输结束时触发一次中断通知处理器。

**18．什么是快表？它在地址转换中起什么作用？**
**快表（TLB, Translation Lookaside Buffer）** 是一种专用且极高速的硬件缓存，内部存储了进程最常使用的页表项（Page table entries）。
**作用：** 虚拟内存分页寻址原本需要先访问主存查页表，再访问主存获取数据。TLB利用硬件级并发比对机制，如果发生命中（TLB Hit），可直接获取物理帧号，免去了前往主存查页表的时间，从而极大地加快了地址转换与内存访问的速度。

**19．什么是死锁？产生死锁的四个必要条件是什么？**
**死锁（Deadlock）：** 是指一组进程永久陷入阻塞状态。这是由于它们各自持有一部分系统资源，又在等待获取组内其他阻塞进程所持有的资源，导致彼此都无法向前推进。
**四个必要条件：**
1. **互斥（Mutual exclusion）：** 一次只有一个进程能使用该资源。
2. **占有并等待（Hold and wait）：** 进程在持有资源的同时等待其他资源。
3. **非抢占（No preemption）：** 资源不能被操作系统强行从持有者手中夺走。
4. **循环等待（Circular wait）：** 存在一个进程等待链，构成了一个闭合的资源需求环。

**20．在哲学家算法中，是否能防止或解除死锁？为什么？**
如果采用天真的信号量解法（每个哲学家都同时拿起左手筷子），会发生死锁。
**可以防止死锁**。常见的防止策略包括：
1. 限制就餐人数：在一个含有5名哲学家的房间中最多只允许4名哲学家同时竞争进餐（使用一个初值为4的信号量），这样至少有1人可以获得两根筷子完成进餐，打破“循环等待”和“占有并等待”的死锁条件。
2. 引入管程（Monitor）：利用管程将叉子封装为状态数组，强制哲学家只有在左右筷子均空闲时才能一次性抓起两根筷子。

**21. 在单处理机环境下，进程间有哪几种通信方式，是如何实现的？**
以经典UNIX系统为例，主要通信方式（IPC）有：
1. **管道（Pipes）：** 基于生产者-消费者模型的环形缓冲区，主要用于相关联的进程间字节流传递。
2. **消息（Messages）：** 使用 `send` 和 `receive` 原语，通过系统内存中的消息队列进行数据结构交换。
3. **共享内存（Shared Memory）：** 操作系统将同一块物理内存映射到不同进程的虚拟地址空间，读写速度最快。
4. **信号量（Semaphores）：** 一个整型变量（含等待队列），通过原子操作 `semWait` 和 `semSignal` 实现进程的互斥与时序同步。
5. **信号（Signals）：** 异步软件中断机制，用于通知进程突发事件的发生。

**22. 在段页式虚拟存储系统中，不同进程之间是如何实现程序共享的？**
在结合了分段与分页优点的系统中，地址空间分为段，段内分为页。由于“段”对应一个逻辑含义完整的实体（如公共库代码），操作系统只需要在不同进程各自的**段表（Segment table）**中，将对应的段表项指向主存中相同的物理段地址（或共享相同的页表），便能安全高效地实现代码或数据的共享。

**23. 在生产者—消费者问题中，能否将生产者进程的wait(empty)和wait(mutex)语句互换，为什么？**
**不能互换**。
如果互换，生产者首先执行 `wait(mutex)` 获得互斥锁，接着执行 `wait(empty)` 申请空闲区。假如当时缓冲区已满（empty=0），生产者将因缺乏空闲位被阻塞。由于它在阻塞时并未释放 `mutex`，此时消费者虽然想消费数据腾出空位，但由于无法获得互斥锁 `mutex` 也将陷入阻塞。双方互相等待对方释放资源，导致了典型的死锁（Deadlock）。

**24. 目前操作系统采用的目录结构是什么？它具有什么优点？**
现代操作系统广泛采用的是**树形结构目录（Tree-structured directory）**。
**优点：** 
1. 有效避免了文件命名冲突（Name clashes），因为不同目录下可以有同名文件；
2. 极大地方便了按所有权、应用程序进行逻辑分类与隔离；
3. 提高了搜索效率、文件的访问安全性以及系统的完整性管理（如按目录组织备份）。

**25. 文件目录有何作用？**
文件目录构成了文件系统中至关重要的数据结构，其作用在于：维护文件的存在状态，记录文件在辅助存储器上的物理位置；包含权限控制信息（Access control element），规定了谁能读、写、执行该文件；同时记录了文件的所有者、创建及修改时间等属性。它是实现文件“按名存取”和安全保护的核心。

**26. 什么是文件的逻辑结构和物理结构？**
*   **逻辑结构：** 站在程序和用户视角，数据是如何被整理的。如无结构的字节流（Pile）、顺序文件、索引文件及散列文件等。
*   **物理结构（磁盘布局）：** 站在操作系统和存储设备的视角，文件内容是如何在物理扇区/盘块中分配和存放的。如连续分配、链式分配和索引分配。

**27. 在操作系统中为什么要引入进程概念？它与程序的差别和关系是怎样的？**
**为什么要引入：** 在多道批处理和分时系统引入后，多程序交替（并发）执行导致了复杂的时序、死锁、同步和状态保存问题。为了能系统地追踪各并发任务的状态和资源，引入了进程（Process）作为操作系统调度和分配资源的基本单位。
**差别与关系：** 程序仅仅是静态存放在辅存中的指令和数据集合（代码实体）；进程是程序的一次动态执行活动（A program in execution）。进程除了包含程序代码和数据外，还附带由操作系统专门维护的“执行上下文/进程控制块（PCB）”。

**28. 什么是进程的互斥与同步？**
*   **进程互斥（Mutual Exclusion）：** 系统中多个进程因争夺同一不可共享的临界资源，而互相排斥的执行关系。当一个进程在临界区内操作时，绝对禁止其他竞争进程进入。
*   **进程同步（Synchronization）：** 多个协作进程为了共同完成某项任务，需要在执行速度或特定执行步骤上互相等待、发信号协调。以确保它们遵循必要的先后发生次序和数据一致性（如生产者需等消费者腾出空间）。

**29. 简述分时系统的四个基本特征？**
1. **多路性（多道/同时性）：** 允许多个联机用户同时通过各自的终端使用一台计算机系统。
2. **交互性：** 用户能够直接通过终端与系统及正在执行的程序进行会话和控制。
3. **独立性：** 每个终端用户在独立的环境中工作，互不干扰，仿佛独占整台计算机。
4. **及时性：** 利用时间片轮转（Time slicing / Round-robin）或抢占调度，系统保证能在极短的响应时间内（Response time）快速反馈用户发出的指令。

**30. 文件的逻辑结构分为几种形式？文件的磁盘布局分为几种形式？**
**逻辑结构：** 堆文件（Pile）、顺序文件（Sequential）、索引顺序文件（Indexed sequential）、索引文件（Indexed）、直接/散列文件（Hashed）。
**磁盘布局（物理分配）：** 连续分配（Contiguous）、链式分配（Chained）、索引分配（Indexed）。

**31. 什么是计算机操作系统？**
操作系统是连接应用软件与计算机底层硬件的系统软件接口（Interface）。它利用一个或多个处理器的资源，负责管理调度CPU、分配主存及辅助存储、控制各I/O设备的运作，并向系统用户和开发者提供安全、一致、丰富的抽象服务（如文件系统、并发进程管理等）。

**32. 简述存储管理中移动技术的优缺点。**
**移动技术（又称压缩/Compaction）**用于在动态分区内存管理中通过内存拷贝将所有活跃进程聚拢到连续块中。
**优点：** 可以彻底消除系统中的“外部碎片”，合并出足够大的连续空闲内存块来容纳新进程，从而提高内存的总体使用率。
**缺点：** 极其耗费处理器时间（搬运大数据块极慢）；还需要系统在硬件层面（基址/限长寄存器）绝对支持进程代码的动态重定位。

**33. 简述并发与并行之间的关系。**
*   **并发（Concurrency）：** 指宏观上多个进程在同一时间段内都在取得进展，但微观上在单处理器系统中可能只是快速的时间片交替运行（Interleaving）。它涉及资源争用、同步通信等设计问题。
*   **并行（Parallelism）：** 指多个进程在多处理器系统（SMP）或多核中，在同一个瞬间真正地同时执行（Overlapping）。并行是实现并发的一种极致物理手段。

**34. 页式和段式内存管理有什么区别？怎样才能实现共享和保护？**
**区别：**
*   **分页（Paging）：** 对程序员透明，系统按机械的固定大小（如4KB）把内存和程序分块，消除了外部碎片，但会产生少量内部碎片。
*   **分段（Segmentation）：** 程序员可见，按具有逻辑意义的模块（函数、变量表）进行不等长划分，无内部碎片，但会产生外部碎片。
**实现共享与保护：** 分段机制在此极具优势。因为一个段包含逻辑上独立的一块内容，只需让多个进程的段表内某一条目指向相同的物理内存基址即可完成“共享”；通过在段表描述符中额外设置权限控制位（只读、读写、仅执行），即可在段的粒度上提供完美的访问“保护”。

**35. 比较基于索引节点和基于符号链的文件共享方法。**
*(注：根据UNIX操作系统的典型文件管理特性 进行系统学比较)*
1. **基于索引节点（Hard link / 目录项直接指向 inode）：** 不同用户的文件目录项（文件名）直接指向同一个物理文件的索引节点（inode）。这种共享速度极快，维护了一个引用计数，且在原链接删除后只要引用数不为0依然可以访问。缺点是通常不能跨越不同的文件系统。
2. **基于符号链（Symbolic link / 软链接）：** 系统创建一个特殊的新文件（符号链接文件），该文件中仅包含所指向共享文件的“路径名”。优点是能够跨越不同的文件系统或网络驱动器实现共享；缺点是每次访问需要多重解析路径名增加开销，且若源物理文件被所有者删除，符号链中的路径指针就会悬空失效。
